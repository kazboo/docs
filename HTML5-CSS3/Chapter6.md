# ボックスと情報の整理

* 前章までのタグは「コンテンツに意味付け」するのが主な役割
* 本章では情報の「グループ化」と「整理」のために使われるものが中心

## インラインボックスとブロックボックス

* ボックス
  * 全てのタグが確保する、コンテンツを表示する領域
  * `インラインボックス`と`ブロックボックス`の2種類がある

* インラインボックス
  * テキストの行に紛れ込むことができるボックスのこと
  * `コンテンツが収まる最小限`のボックスを形成する
    * `<b>, <strong>` など

* ブロックボックス
  * `タグに含まれるコンテンツの量に関係なく`、`親要素と同じ幅の領域を確保する`タイプのボックス
  * さらに大きく以下の2つに分類される
    * そのタグがコンテンツに何らかの意味付けをするもの
      * 見出しの`<h1>`、段落の`<p>` 等
      * 多くはテキストを装飾するタイプのタグしか含めることができない
    * 他のタグを囲んで、情報の整理をしたりグループ化したりするもの
      * 箇条書きの`<ul> <li>`、ほかのタグをグループ化する`<div>` 等
      * どんなタグでも子要素にできる

* インラインボックスとブロックボックス
  ```html
  <p>
      <b>hoge:</b> huga

  </p>
  ```
  ```txt
  [[hoge:] huga] - ブロックボックス
      |
  インラインボックス
  ```
  タグは以下のように分類される
  ```txt
  [要素]
    +--インラインボックス(img, input, strong, a, ...)
    |
    +--ブロックボックス
        |
        +---コンテンツに意味付けをするボックス(p, h1, ...)
        |
        +---情報を整理・グループ化するボックス(div, ul, li, footer, ...)
  ```

## 箇条書き(リスト)のマークアップ

* `非順序リスト(unordered list)`
  * 全体的に使用頻度が高いリスト関連のタグの中でも、最もよく使われる
  * `<ul>, <li>`
    * リストの各項目のコンテンツが同じような意味合いを持つ場合に使われる(重要度、順序に差がない)
    * `ul`の子要素は必ず`li`で、ほかのタグが`ul`の`直接の`子要素になることはない

* `順序リスト(ordered list)`
  * 操作手順など、各項目に順序や序列がある場合
  * `<ol>, <li>`

* `ul`の用途
  * ナビゲーション
    * サイト内の主要なページにリンクする
  * サイトマップ
    * フッターに掲載され、サイト内の各ページにリンクする
  * おしらせ
  * 商品一覧
  * 最新記事一覧
    * 記事、ニュース系のサイトやブログで掲載される

* example
  * サブナビゲーションのマークアップ例
  * サブナビゲーションの先頭にマークをつける
  * サブナビゲーションの先頭に画像を表示させる典型的なCSS
  * パンくずリストを作成する

* パンくずリストをつくるなら`<ol>`
  * 階層構造を表現するため、順列があるolのほうが適切

* 要素のボックスの表示状態を変える`displayプロパティ`
  * liはもともとブロックボックス > インラインボックスとして表示させる(block > inline)
  * displayプロパティの主な値
    * inline: 要素をインラインボックスで表示する
    * inline-block: 要素をインラインブロックで表示する
    * block: 要素をブロックボックスで表示する
    * none: 要素を非表示にする

* インラインブロック
  * インラインボックスのようにテキストの行に紛れ込むことができます
  * さらに上下マージンや幅を設定することもできる
  * floatプロパティを使わずにブロックボックスを横に並べたいときに使用する

* `:last-child`セレクタ
  * 最後の`li`要素だけ選択する（その要素にだけ`≫`を表示しないようにしている)
  * 同階層の要素のうち、最後のものだけを選択するセレクタ


## 上手なdivの使い方

* div
  * 他の要素をグループ化するためのタグで、それ自身は意味を持たない
  * 一旦デザインのことを忘れて、できるだけ情報のまとまりを意識しながら囲むようにするといい
    * 最終的なデザインの見た目に合わせてCSSを適用させるために囲むケースが多い
    * ソースコードにやたらとdivが増えてくる
    * 階層構造が深くなり、HTMLは読み辛く、メンテナンス性が低下する
    * 不要なdivはなるべくなくしていく

* おおよそ4つのパターンに分けられる
    1. 見出しと関連するコンテンツをまとめる
    1. パーツの境界を作る
    1. HTMLの階層をそろえる
    1. 別のボックスのラッパー構造を作る

### 見出しと関連するコンテンツをまとめる

```html
<div class="spec">
    <h2>基本仕様</h2>
    <ul>
        <li>サイズ: 24.5 x 28.0cm x 32.4cm</li>
        <li>重量: 6.5kg</li>
    </ul>
</div>
```

* HTML5で登場した、section等で置き換えることができる

### パーツの境界を作る

* サイドバー
  * ページ内検索、お知らせ、最新記事、広告等
  * 独立した情報をパーツと考え、境界をはっきりさせるためdivで囲む
  * パーツの順序入れ替えが楽になる

```html
<div class="ad">
    <img src="koukou.jpg" alt="アクセス解析ならオートアナライズ">
</div>
<div class="top3">
    <h3>売上トップ3<h3>
    <ul>
        <li>ソウルフードストラップ10点セット</li>
        <li>魚眼レンズ付きスマートフォンケース</li>
        <li>だれでも貼れるガラス保護シート</li>
    </ul>
</div>
```

### HTMLの階層をそろえる

* 美しくないHTML
  ```html
  <body>
      <!-- header部 -->
      <div class="header">...</div>
      <!-- メインコンテンツ -->
      <h1>...</h1>
      <p>...</p>
      <!-- footer部 -->
      <div class="footer">...</div>
  </body>
  ```

* 美しいHTML
  ```html
  <body>
      <div class="header">...</div>
      <div class"main">
          <h1>...</h1>
          <p>...</p>
      </div>
      <div class="footer">...</div>
  </body>
  ```
  兄弟要素が同じカテゴリーに属している

### 別のボックスのラッパー構造を作成する

* ラッパー構造
  * divで囲まれた複数のグループ化された構造を、さらにdivで囲むこと
  * 主にCSSで複数のdivを横に並ばせたい時等に使われる

```html
<div class="wrapper">
    <div class="main">...</div>
    <div class="sidebar">...</div>
</div>
```

### 慣れてきたら、div以外のより的確なグループ化要素に置き換える

* HTML5になりほかの要素をグループ化し、それ自身が明確な意味を持つものが登場した
  * 慣れてきたら置き換えてみるとよい

* 意味を持つ情報のグループ化用タグ
  * `<main>...</main>`
    * ページの中心となるメインコンテンツを囲む
  * `<article>...</article>`
    * ページの記事や中心となるコンテンツを囲む
  * `<section>...</section>`
    * 記事のセクションや、ページ内の独立したパーツを囲む
  * `<nav>...</nav>`
    * ページの主要なナビゲーションを囲む(一般的にはグローバルナビゲーション)
  * `<aside>...</aside>`
    * ページの本棚ではない部分、例えばサイドバーなどを囲む
  * `<header>...</header>`
    * ページのヘッダーを囲む
  * `<footer>...</footer>`
    * ページのフッターを囲む

## CCSのボックスモデル

* ボックスモデル
  * ボックス：個々のタグがコンテンツを表示するために確保する領域のこと
  * 全てのボックスの中心にはコンテンツを表示するための`コンテンツ領域`がある
  * その周りにパディング、ボーダー、マージンという領域がある
  ```
  +-----------------------------------------+
  |                 margin                  |
  |    +------------border-------------+    |
  |    |  +-------------------------+  |    |
  |    |  |         padding         |  |    |
  |    |  |    +---------------+-----------------
  |    |  |    |               |    |  |    |  |
  |    |  |    |    content    |    |  |    | height
  |    |  |    |               |    |  |    |  |
  |    |  |    +---------------+-----------------
  |    |  |    |               |    |  |    |
  |    |  +----|---------------|----+  |    |
  |    +-------|---------------|-------+    |
  |            |               |            |
  +------------|---------------|------------+
               |----width------|
  ```

### ボックスの幅と高さ

* ブロックボックス(p, divなど)のコンテンツ領域のサイズ(CSS適用なし)
  * 幅：親要素の幅から、`マージン・ボーダー・パディングがはみ出さない範囲でいっぱいに広がる`
    * デフォルトCSSが`width:auto`であるため
  * 高さ：コンテンツが収まる高さになる
    * `上下マージンは親要素のボーダーの外に出る`
  * ブロックボックスのコンテンツ領域はwidth, heightを指定することができる
    * 使われることはあまり多くない

* インラインブロックのコンテンツ領域のサイズ
  * その要素のコンテンツが収まるギリギリの大きさになる
  * 幅も高さもCSSで調整することは基本できない
    * ※ img, input, textareaなど一部の要素は幅や高さをCSSで調整できる

### パティング、ボーダー、マージン

* パディング
  * コンテンツ領域のすぐ外側にあって、コンテンツ領域とボーダーの間にあるスペースのこと

* ボーダー
  * ボックスの周囲に惹かれる枠線

* ボックスに背景色、背景画像を指定した際は、パディングの領域までが塗りつぶされる 

* マージン
  * あるボックスの上下左右に隣接する別のボックスまでの距離のこと
    * 「これ以上近づかないで」というバリアのような役目
  ```
  +-----------+  +-----------+
  |    box A  |--|    box    |
  +-----------+  +-----------+
        |     box A のマージン(隣接するボックスとの距離)
  +-----------+
  |    box    |
  +-----------+
  ```
  * `インラインボックスには上下マージンを設定することができない`
    * 左右マージン、四辺のパディング、ボーダーは設定できる

* ショートハンドとロングハンドプロパティ(CSSプロパティ)
  * ショートハンド(省略形)プロパティ
    * 複数の設定を一括でできる
      * ex)padding
  * ロングハンド(非省略形)プロパティ
    * １つの設定につき１つのプロパティ
      * ex)padding-top, padding-left
  * 好きなほうを使ってよい（一般的には記述量が少なくなるショートハンドが使われることが多い)


## パディング、ボーダーの設定

* ボックスに枠線をつける
  ```html
  <style>
  .item {
      /* ここ */
      border: 1px solid #cccccc;
      width: 300px;
  }
  .item h3 {
      margin: 0.5em 0 0.5em 0;
      font-size: 16px;
  }
  .item p {
      margin: 0;
      font-size: 14px;
  }
  </style>
  </head>
  <body>
      <div class="item">
          <img src="../../images/img0907.jpg" width="300" height="300" alt="">
          <h3>限定　無人島探検チャーター便</h3>
          <p>ほげ</p>
      </div>
  </body>
  ```

* borderプロパティ
  ```css
  border: 太さ 線の形状 色;
  ```
  * 太さは単位が必要、pxがほとんど
  * 線の形状
    * none：太さを指定しても確保されない
    * dotted：点線、四辺だと角が汚くなるので、一般的には一辺に適用するときに使う
    * dashed：長めの点線、四辺だと角が汚くなるので、一般的には一辺に適用するときに使う
    * solid：実践
    * double：二重線

* borderを区切り線のように使う
  ```css
  .item {
      border-right: 1px solid #cccccc;
      border-bottom: 1px solid #cccccc;
      width: 300px;
  }
  ```

* borderの全プロパティ
  * border-right
  * border-left
  * border-top
  * border-bottom
  * border

* コンテンツと枠線の間にパディングを設ける
  * ボーダーだけを設定しても、コンテンツとの間にスペースがないので見た目が美しくない
  * paddingプロパティでスペースを作る
  ```css
  .item {
      border: 1px solid #ccc;
      padding: 8px;
      width: 300px;
  }
  ```

* paddingプロパティ
  ```css
  padding: 上 右 下 左;
  ```
  * 時計回り
  * 省略できる
    ```css
    /* [上右下左] */
    padding: 8px;
    /* [上下]　[右左] */
    padding: 8px 4px;
    /* [上] [右左] [下] */
    padding: 8px 4px 6px;
    /* [上] [右] [下] [左] */
    padding: 8px 4px 6px 2px;
    ```
  * borderプロパティ同様
    * padding-top
    * padding-right
    * padding-bottom
    * padding-left
    * padding
